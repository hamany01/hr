# وثيقة متطلبات المنتج (PRD) - منصة التوظيف الذكية

## 1. نظرة عامة على المشروع
منصة توظيف احترافية متكاملة مصممة خصيصاً للشركات لإدارة عمليات التوظيف وفرز المرشحين باستخدام تقنيات الذكاء الاصطناعي (AI). تتميز المنصة ببنية تحتية مرنة (Supabase) وواجهة مستخدم باللغة العربية بالكامل، مع التركيز على تقليل التكاليف التشغيلية (API Tokens) عبر دمج حلول الذكاء الاصطناعي المحلية.

---

## 2. هيكلة المستخدمين والصلاحيات

### 1. المرشح (Candidate)
* التقديم على الوظائف المتاحة.
* رفع السيرة الذاتية (PDF, Word, Images).
* الإجابة على الأسئلة المخصصة للوظيفة (بحد أقصى 20 سؤال).
* تتبع حالة الطلب وتلقي الإشعارات.

### 2. مدير القسم (Department Manager)
* استعراض المرشحين المتقدمين لوظائف قسمه فقط.
* الاطلاع على التقييم الآلي للذكاء الاصطناعي (AI Score & Ranking).
* تقديم توصيات (قبول مبدئي، طلب مقابلة، رفض).

### 3. مدير الموارد البشرية (HR Manager)
* الإنشاء والإدارة الكاملة للوظائف.
* رفع ملفات الأسئلة (JSON/MD) التي تم توليدها خارجياً.
* إدارة رحلة المرشح وإرسال إشعارات الواتساب.
* اتخاذ قرارات التوظيف النهائية.

### 4. مدير النظام (System Admin)
* إدارة الصلاحيات للمستخدمين والموظفين.
* إعدادات المنصة الأساسية.

---

## 3. إدارة الوظائف والأسئلة (الاستيراد الديناميكي)

لتوفير تكاليف الذكاء الاصطناعي داخل المنصة، تم اعتماد آلية هجينة لإنشاء الوظائف:
1. **البيانات الأساسية:** يقوم الـ HR بإدخال (المسمى الوظيفي، الوصف، المتطلبات، الراتب، نوع الدوام) عبر لوحة التحكم.
2. **توليد الأسئلة خارجياً:** توفر المنصة "دليل هندسة أوامر" (Prompt Manual) يقوم الـ HR بنسخه ولصقه في منصات مثل (ChatGPT/Claude). يسأل الذكاء الاصطناعي الـ HR عن تفاصيل الوظيفة ويقوم بتوليد ملف `JSON` أو `MD` يحتوي على الأسئلة (حتى 20 سؤال).
3. **استيراد الأسئلة:** يتم رفع الملف للمنصة، وتقوم المنصة باستخراج الأسئلة وحفظها مباشرة في **قاعدة البيانات** (Database)، ثم يُحذف الملف لتتمكن الإدارة من التعديل أو الحذف من لوحة التحكم مباشرة.

---

## 4. دور الذكاء الاصطناعي (AI Capabilities)

### 1. معالجة السير الذاتية (Local AI Worker) - مجاني التكلفة
* لتفادي استهلاك (Tokens) سحابية ضخمة، سيتم تطوير **وكيل ذكاء اصطناعي محلي (Python Worker)** يعمل على أجهزة الشركة أو خادم افتراضي رخيص (VPS).
* يعتمد الوكيل على نظام طابور مهام مخزن بالكامل في قاعدة البيانات (Database-backed Job Queue) لضمان موثوقية تشغيلية عالية ومصدر وحيد للحقيقة.
* للتفاصيل الفنية الكاملة وهيكل قاعدة البيانات ومخططات الجداول، يرجى الرجوع لملف التصميم المستقل: [`PYTHON_WORKER_ARCH_AR.md`](./PYTHON_WORKER_ARCH_AR.md).

### 2. التقييم الذكي واستراتيجية التبديل التلقائي (AI Routing & Fallback Strategy)
* **إدارة مفاتيح واجهة برمجة التطبيقات (API Keys Database):** يتم تخزين مجموعة من مفاتيح API السحابية (المجانية أو المدفوعة) في قاعدة البيانات (Database).
* **آلية التبديل التلقائي (Sequential Fallback):** عند الحاجة لتقييم مرشح، يقوم النظام باستدعاء المفتاح الأول. إذا تعطل أو استنفد رصيده (مثلاً خطأ 429 أو 401)، ينتقل النظام تلقائياً للمفتاح الذي يليه في القائمة.
* **التحويل للخادم المحلي (Local AI Fallback):** في حال تعطلت جميع مفاتيح الـ API السحابية، يقوم النظام آلياً بتحويل عملية التقييم إلى "الخادم المحلي" (Ollama / Python Worker) لتنفيذ المهمة دون توقف الخدمة.
* **هندسة الأوامر المتعددة (Multi-Persona Prompting):** يعتمد التقييم على تقمص عدة شخصيات (خبير موارد بشرية، مدير فني، محلل نفسي) لضمان جودة التقييم.
* يقتصر مخرج الذكاء الاصطناعي على: إعطاء درجة (Score)، ترتيب المرشحين (Ranking)، ونقاط القوة والضعف.

## 5. رحلة المرشح والتواصل (WhatsApp Integration)

تعتمد المنصة حالياً على الروابط المباشرة (wa.me) مع رسائل جاهزة (Templates)، قابلة للترقية مستقبلاً إلى (Meta WhatsApp API).
تمر رحلة الإشعارات بالمراحل التالية:
1. **تأكيد التقديم:** رسالة شكر على التقديم للوظيفة المحددة.
2. **دعوة للمقابلة:** رسالة تحتوي على تفاصيل وموعد المقابلة.
3. **دخول المقابلة:** تذكير وقت المقابلة (أونلاين أو حضوري).
4. **شكر بعد المقابلة:** رسالة شكر على الحضور، وتوضيح أن الرد النهائي سيكون خلال (5 إلى 9 أيام عمل) لفرز المرشحين.
5. **القرار النهائي:** رسالة اعتذار احترافية، أو رسالة قبول مبدئي لطلب مسوغات التعيين.

---

## 6. البنية التحتية والتقنيات المقترحة (Tech Stack)

* **لغة الواجهة (UI/UX):** عربية (RTL) بالكامل، تصميم احترافي مخصص لكل قسم.
* **الواجهة الأمامية (Frontend):** React.js (Vite) + Tailwind CSS.
* **قاعدة البيانات والمصادقة (Backend/DB):** Supabase (PostgreSQL, Auth, Storage).
* **وكيل الذكاء الاصطناعي المحلي (Local AI):** Python, FastAPI, Local LLMs (Ollama).
* **التكامل السحابي (Cloud AI):** OpenAI API أو Gemini API (لتقييم الإجابات فقط وترتيبها).
